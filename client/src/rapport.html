Rapport de Projet

Application de Cartographie Interactive - Nancy

Contexte

Ce projet propose une application web interactive permettant de visualiser sur une carte les stations de vélos en libre-service, les restaurants et les incidents signalés dans la ville de Nancy. L’application vise à faciliter la mobilité urbaine et l’accès à l’information en temps réel.

Technologies utilisées

  • Frontend : JavaScript, HTML5, CSS3, Bootstrap 5, Leaflet
  • Backend : Java (Spring Boot), Maven
  • APIs externes : Cyclocity (GBFS), BAN (Base Adresse Nationale), OpenStreetMap
  • Outils : npm, IntelliJ IDEA

Fonctionnalités principales

  • Affichage dynamique des stations de vélos avec disponibilité en temps réel
  • Visualisation et réservation de restaurants sur la carte
  • Affichage des incidents de circulation
  • Géolocalisation de l’utilisateur et calcul d’itinéraires (vers station ou restaurant)
  • Statistiques globales (nombre de vélos, places libres, etc.)
  • Interface responsive et ergonomique

Architecture générale

  • Frontend :
    • index.html : structure de la page et intégration des composants
    • script.js : logique de la carte, gestion des données, interactions utilisateur
    • config.js : centralisation des URLs et paramètres de configuration
  • Backend :
    • API REST pour la gestion des restaurants et des réservations
    • Proxy pour les données incidents et agrégation des sources externes

Points techniques notables

  • Utilisation de Leaflet pour la cartographie et la gestion des marqueurs personnalisés
  • Appels asynchrones aux APIs pour garantir la réactivité
  • Gestion des erreurs et affichage de modales Bootstrap pour l’expérience utilisateur
  • Animation progressive des marqueurs pour une meilleure lisibilité
  • Réservation intelligente avec proposition de créneaux alternatifs en cas d’indisponibilité

Conclusion

Ce projet démontre l’intégration de multiples sources de données et la création d’une interface cartographique moderne, intuitive et utile pour les citoyens. Il peut être enrichi par d’autres services urbains ou adapté à d’autres villes.
